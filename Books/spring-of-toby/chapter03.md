# 템플릿

## 템플릿 메소드 패턴
- 템플릿 메소드 패턴은 상속을 통해 기능을 확장해서 사용하는 패턴이다.
- 변하지 않는 부분은 슈퍼클래스에 두고 변하는 부분은 추상 메소드로 정의하여 서브클래스에서 오버라이드하여 재정의하여 쓰는 것.
- 단점은 공통로직을 새로운 부분에 적용하기위해 새로운 클래스를 만들어 상속받아야 한다는 것.

## 전략 패턴
- 전략 패턴은 '변하는 부분'을 별도의 인터페이스로 정의하고, 이를 구현한 클래스를 '변하지 않는 부분'에 주입한다.
- OCP(개방 폐쇄 원칙)를 준수하는 깔끔한 구조가 완성
- Context는 전략 인터페이스에만 의존하므로, 구체적인 전략이 바뀌어도 코드를 수정할 필요가 없다.

## 템플릿과 콜백
- 스프링은 전략 패턴을 기본으로 하되, 익명 내부 클래스를 활용한 템플릿/콜백 패턴을 사용.
- 템플릿은 고정된 작업 흐름을 가진 코드를 재사용한다는 의미에서 붙인 이름이다.
  - 템플릿: 전략 패턴의 컨텍스트. 고정된 작업 흐름을 담당
- 콜백은 템플릿 안에서 호출되는 것을 목적으로 만들어진 오브젝트를 말한다.
  - 콜백: 전략 패턴의 전략. 템플릿 안에서 호출되는 익명 함수 같은 역할을 담당  
  실행되는 시점에 구체적인 로직을 담아 템플릿에게 전달
- 매번 클래스 파일을 만들 필요 없이, 메소드 내부에서 간결하게 전략을 정의하여 주입할 수 있다.

## 중첨 클래스
- static class: 독립적인 클래스로 존재하며, 바깥 클래스의 인스턴스와 관계가 없다
- inner class: 바깥 클래스의 인스턴스에 종속
  - member inner class: 클래스 내부에 멤버처럼 정의
  - local inner class: 메소드 내부에 정의
  - anonymous inner class: 이름 없이 정의와 생성을 동시에 함

### JdbcTemplate
- 템플릿/콜백 패턴을 사용해서 스프링은 JdbcTemplate을 구현.
- 개발자는 더 이상 try-catch-finally와 리소스 관리에 신경 쓸 필요가 없다.
- DB 연결 관리 같은 저수준 관심사와 높은 수준의 비즈니스 관심사(데이터 처리)가 완벽하게 분리

### 핵심
- 변하는 것과 변하지 않는 것을 분리하고, 인터페이스를 통해 느슨하게 연결하여, 변경에는 닫혀 있고 확장에는 열려 있는 유연한 코드를 만드는 것.
